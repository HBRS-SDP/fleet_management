Task planner¶

class fleet_management.plugins.planning.TaskPlannerInterface(kb_database_name, domain_file, planner_cmd, plan_file_path, **_)¶

An interface for generating ROPOD task plans.

Code author: Alex Mitrevski <aleksandar.mitrevski@h-brs.de>

_get_location_floor(location)¶

Return the floor number of a given location. For ROPOD, this can either be done through the OSM path planner or by parsing an Area string

Parameters

location – An Area string

Returns

The floor number of an area

Return type

floor (int)

_get_task_plan_without_robot(task_request: ropod.structs.task.TaskRequest, path_planner)¶

Generates a task plan based on the given task request and returns a list of ropod.structs.action.Action objects representing the plan’s actions

Parameters

  • task_request – task request parameters

  • path_planner – an interface to a path planner used for planning paths once a task plan is obtained

_plan_paths(task_plan: list, path_planner)¶

Plans paths between the areas involved in the task plan. Returns the list of task actions in task_plan with added paths between the areas involved in the plan.

Parameters

  • task_plan (list) – a list of ropod.structs.action.Action objects

  • path_planner – an interface to a path planner

plan(request, path_planner)¶

Temporary solution to translate between the TaskRequest model and the existing TaskRequest struct
