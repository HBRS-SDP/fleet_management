health_status_cb(msg)¶

Process the HEALTH-STATUS messages sent by the robot

Since there is no summarized version of the health status message the callback processes the detailed version sent by the component monitoring. This makes a few, very simplistic assumptions about whether a robot is defective or not:

  • If any component is faulty, we mark the robot as FAILED

  • If all components are working correctly, we mark the robot as OPTIMAL

This does not have the nuance of the degraded or suboptimal performance of the robot, but is in line with the current version of the component monitoring and the remote monitoring interface.

msg – The zyre message as a dictionary.
